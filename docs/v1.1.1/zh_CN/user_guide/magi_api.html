灵活 Flash Attention#

为了支持不规则形状掩码的计算，我们实现了 flexible_flash_attention 内核，可通过以下接口调用。

分发（Dispatch）#

变长分发（Varlen Dispatch）#

如果您使用的是由 cu_seqlens 定义的掩码（如 varlen full 或 varlen causal 掩码），我们参照 FlashAttention 的 API 设计了类似的接口 magi_attn_varlen_key，方便您快速上手。

如果您想在同一训练步中应用多个掩码，可以使用 make_varlen_key_for_new_mask_after_dispatch 为新掩码创建密钥，只需提供 flash-attn-varlen 风格的 varlen 掩码参数和已有的 dispatch 密钥。

新掩码将复用与 dispatch 掩码相同的分发方案，但使用不同的元参数进行计算和通信。

灵活分发（Flexible Dispatch）#

如果您使用的掩码不限于 varlen full 或 varlen causal，还包括滑动窗口掩码或其他更多样的类型，我们建议使用 magi_attn_flex_key，如下所示。

如果您想在同一训练步中应用多个 varlen 掩码，可以使用 make_flex_key_for_new_mask_after_dispatch 为新掩码创建密钥，只需提供掩码参数和已有的 dispatch 密钥。

新掩码将复用与 dispatch 掩码相同的分发方案，但使用不同的元参数进行计算和通信。

分发函数#

获取 dist attn 运行时密钥后，您可以调用 dispatch 函数将全局输入张量沿 seqlen 维度分发，得到填充后的本地张量。

计算注意力#

在 dispatch 和 QKV 投影之后，您将获得本地的 query、key 和 value。然后可以使用 dist attn 运行时密钥调用 calc_attn 来计算分布式注意力，获取本地注意力输出张量。

Roll#

Roll 函数#

在 dispatch 之后，如果您需要沿序列维度循环移位本地张量，可以使用 dist attn 运行时密钥调用 roll。此功能主要为 Multi-Token Prediction (MTP) 设计，其中标签需要相对于输入 token 移位一个或多个位置。它也可用于其他场景，如相对位置偏移或 shifted-window 模式。

语义上，roll 等价于 undispatch -> torch.roll -> dispatch，但避免了实体化完整的全局张量，将峰值内存从 O(N) 降低到 O(N/P)，并将通信量减少约 P 倍。

反分发（Undispatch）#

反分发函数#

当您需要从本地张量恢复全局输出张量（用于计算损失或其他原因）时，可以调用 undispatch 函数将填充后的本地输出张量沿 seqlen 维度反分发回未填充的全局张量。

工具函数#

计算填充大小#

在使用 MagiAttention 时，我们将 total_seqlen 划分为多个大小为 chunk_size 的块，并均匀分配到多个 GPU 上。为确保 total_seqlen 能被 chunk_size 整除，且每个 GPU 接收相同数量的块，我们需要对原始输入进行填充。

您可以调用 compute_pad_size 计算所需的填充长度，并将此值作为后续函数的参数使用。

获取 pad_size 后，您可以使用 pad_at_dim 和 unpad_at_dim 函数对张量进行填充和去填充操作。

类似地，您可以使用 pad_size 以及 total_seqlen 和其他相关信息，通过 apply_padding 函数对 (q_ranges, k_ranges, mask_types) 元组应用填充。

此函数使用无效切片填充填充区域。

获取位置 ID#

由于 MagiAttention 需要沿 seqlen 维度对输入张量进行排列变换，一些与 token 相关的操作可能会受到影响，如 RoPE。

因此，我们提供了 get_position_ids 函数来获取输入张量的位置 ID，类似于 Llama。

获取最近的 Key#

如果您在访问元密钥时遇到困难，同时需要获取某个 cp_group 的最新密钥，可以通过指定 cp_group 调用 get_most_recent_key 来获取。

但是，我们强烈建议您通过参数传递的方式访问密钥，以避免意外的不一致性。

推断变长掩码#

如果您想使用每个分段长度相同的 varlen 掩码，我们提供了 infer_varlen_mask_from_batch 函数为您生成相应的 cu_seqlens 张量。

在使用 varlen 掩码时，通常需要将形状为 [batch_size × seq_len, ...] 的张量重塑为 [batch_size × seq_len, ...]。

为便于使用上述 API，我们提供了 squash_batch_dim 函数来合并张量维度。

此外，如果您已经计算了 cu_seqlens 张量并想基于它生成 varlen 掩码，我们提供了 infer_attn_mask_from_cu_seqlens 函数。此函数可以根据 cu_seqlens、causal、window_size 和 global_window_size 创建四种类型的掩码——varlen full、varlen causal、varlen sliding window 和 varlen sliding window with global attention，并以 (q_ranges, k_ranges, mask_types, total_seqlen_q, total_seqlen_k) 的形式返回结果。

当 global_window_size 设为正整数时，样本中的每个 query 除滑动窗口外，还会关注该样本的前 global_window_size 个 key token，这对于需要某些前缀 token（如系统提示、sink token）全局可见的架构非常有用。为防止信息泄露，相对位置 i 处的 query 只能看到位置 [0, min(global_window_size, i + window_size_right + 1)) 的全局 token。

推断滑动窗口掩码#

在 MagiAttention 的设计中，我们使用 (q_range, k_range, masktype) 元组来表示一个切片。

对于滑动窗口掩码，我们没有提供专用的 masktype 来直接表示。

但滑动窗口掩码可以分解为现有 masktype 的组合，如 full、causal、inv_causal 和 bi_causal。

如果您不确定如何进行分解，我们提供了 infer_attn_mask_from_sliding_window 函数来为您处理。
