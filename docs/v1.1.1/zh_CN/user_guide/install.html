安装#

警告

MagiAttention 目前仅支持 Hopper 和 Blackwell。我们正在积极开发对更多 GPU 架构的支持，将在后续版本中推出。

  • 设置环境

    • 启动 NGC-PyTorch 容器

    • 拉取源代码

    • 启用 IBGDA（可选）

  • 设置依赖

    • 安装必需包

    • 安装 flash_attn_cute（可选）

  • 安装 MagiAttention

    • 从源码安装 MagiAttention

    • 预编译 FFA_FA4 内核（可选）

设置环境#

启动 NGC-PyTorch 容器#

小技巧

我们建议您使用标准的 NGC-PyTorch Docker Releases 以保持 Python、CUDA、PyTorch 等基础依赖的一致性。

警告

由于 CUDA-12 导致的性能问题，我们建议您使用基于 CUDA-13+ 的 NGC-PyTorch 容器以获得最佳性能。

我们在 setup.py 脚本中添加了断言来检查 CUDA 版本，如果 CUDA 版本低于 13.0 则会中止安装。

如果您坚持使用基于 CUDA-12 的容器，可以设置环境变量 MAGI_ATTENTION_ALLOW_BUILD_WITH_CUDA12=1，但请注意这可能会导致与 CUDA-13+ 相比显著的性能下降。

  • docker run 命令：

    # choose one compatible version
MAJOR_VERSION=25
MINOR_VERSION=10

# specify your own names and paths
CONTAINER_NAME=...
HOST_MNT_ROOT=...
CONTAINER_MNT_ROOT=...

docker run --name ${CONTAINER_NAME} -v ${HOST_MNT_ROOT}:${CONTAINER_MNT_ROOT} -it -d --privileged --gpus all --network host --ipc host --ulimit memlock=-1 --ulimit stack=67108864 nvcr.io/nvidia/pytorch:${MAJOR_VERSION}.${MINOR_VERSION}-py3 /bin/bash

  • docker exec 命令：

    docker exec -it ${CONTAINER_NAME} /bin/bash

拉取源代码#

  • git 命令：

    git clone https://github.com/SandAI-org/MagiAttention.git

cd MagiAttention

git submodule update --init --recursive

启用 IBGDA（可选）#

备注

如果您想在 cp_size > 8 时使用我们的 native group-collective kernels 作为通信后端，即涉及节点内（通过 NVLink 连接）和节点间（通过 RDMA 可见）的进程组，您需要在裸金属主机上启用 IBGDA。

警告

此步骤需要在裸金属主机操作系统上执行，不能在 Docker 或其他容器化环境内执行，因为容器无法管理主机内核。

  • bash 脚本：

    bash scripts/enable_ibgda_on_host.sh

设置依赖#

安装必需包#

  • pip install 命令：

    pip install -r requirements.txt

安装 flash_attn_cute（可选）#

备注

如果您想在 Ampere 或 Blackwell 上试用 MagiAttention，目前需要安装 flash_attn_cute 包以启用 FFA_FA backend 作为临时解决方案。

  • bash 脚本：

    ARCHS="sm80,sm100" # if you only want to use in Blackwell, you can set it to "sm100" to speed up the installation; and if you only want to use in Ampere, you can set it to "sm80" accordingly

bash scripts/install_flash_attn_cute.sh $ARCHS

安装 MagiAttention#

从源码安装 MagiAttention#

警告

此过程首次可能需要约 10~20 分钟，并占用 90% 的 CPU 资源。

备注

我们提供了若干环境变量来细粒度控制安装过程，尤其是 CUDA 扩展模块的构建。

  • Hopper 的 pip install 命令：

    pip install --no-build-isolation .

  • Blackwell 的 pip install 命令：

    export MAGI_ATTENTION_PREBUILD_FFA=0 # for now, native ffa does not support Blackwell
pip install --no-build-isolation .

export MAGI_ATTENTION_FA4_BACKEND=1 # always set it when using MagiAttention on Blackwell

  • Ampere 的 pip install 命令：

    export MAGI_ATTENTION_PREBUILD_FFA=0 # for now, native ffa does not support Blackwell
export MAGI_ATTENTION_SKIP_MAGI_ATTN_COMM_BUILD=1 # for now, magi_attn_comm does not support Ampere
pip install --no-build-isolation .

export MAGI_ATTENTION_FA4_BACKEND=1 # always set it when using MagiAttention on Ampere

预编译 FFA_FA4 内核（可选）#

备注

如果您想在 Blackwell 上试用 MagiAttention，并且已经安装了 magi_attention 和 flash_attn_cute 以启用 FFA_FA backend，我们进一步建议您在生产使用前预编译 FFA_FA4 内核的常见配置，以避免运行时 JIT 重新编译的开销，因为它基于 Cute PythonDSL 构建。

预编译内核的缓存目录默认为 /path/to/magi_attention/lib/ffa_fa4_cache/，如有需要，可通过设置环境变量 MAGI_ATTENTION_FFA_FA4_CACHE_DIR 来指定自定义缓存目录。

  • python 脚本：

    # You can change the cases to pre-compile in the script according to your needs,
# and the whole pre-compilation progress will be richly logged
# in the terminal by tqdm, for you to track the progress and results.
python tools/precompile_ffa_fa4.py
