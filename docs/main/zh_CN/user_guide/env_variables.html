环境变量#

以下是一些可以设置的环境变量及其说明。

备注

由于 MagiAttention 正在积极发展中，许多高级但实验性的功能将通过环境变量启用后发布。

正确性相关#

MAGI_ATTENTION_FORWARD_HIGH_PRECISION_REDUCE

将此环境变量设为 1 以启用 dist-attn 前向传播中 partial out 的高精度（fp32）reduce，以双倍通信开销换取更高精度和更少的 dtype 转换开销。默认值为 0。

备注

  1. 在 ffa 前向内核内部，我们始终对 partial out 使用高精度（fp32）累加。

  2. 我们在所有地方始终使用高精度（fp32）的 lse。

  3. 此功能仅在启用 qo comm 时对 out 生效。

MAGI_ATTENTION_BACKWARD_HIGH_PRECISION_REDUCE

将此环境变量设为 1 以启用 dist-attn 反向传播中 partial dq,dk,dv 的高精度（fp32）reduce，以双倍通信开销换取更高精度和更少的 dtype 转换开销。默认值为 0。

备注

  1. 在 ffa 反向内核内部，我们始终对 partial dq,dk,dv 使用高精度（fp32）累加。

  2. 此功能仅在启用 qo comm 时对 dq 生效。

MAGI_ATTENTION_DSINK_ALL_REDUCE_OP

设置此环境变量的值以控制涉及 attention sink 时 dist_attn_func 中 sink 梯度的 all-reduce 操作。默认值为 none。可选值为 {none、sum、avg}。

备注

目前我们仅接受全局复制的 sink 张量作为输入传入 dist_attn_func，每个 cp rank 中的 sink 梯度是部分的，需要在 cp rank 之间进行 sum-reduce。

然而，由于 sink 张量是可学习的，它将被视为模型中的常规参数，类似于 nn.Linear 层中的 bias。

因此在一些流行的训练框架（如 Megatron-LM、FSDP）中，sink 部分梯度在 cp rank 之间的 sum-reduction 可能会在整个 dp x cp mesh 中自动执行。

为避免重复 reduction，我们提供此环境变量来指定 dist_attn_func 中 sink 梯度的 all-reduce 操作，其默认值为 none，即 dist_attn_func 不对 sink 梯度执行任何 reduction，而由框架来处理。

但在没有任何框架机制来跨 cp rank reduce 参数的场景下，您必须将此环境变量设为 sum。

有时 avg 也是一个选项，当您需要将 sink 梯度按 1/cp 缩放时可以使用。

性能相关#

MAGI_ATTENTION_FA4_BACKEND

将此环境变量设为 1 以将注意力内核后端从 FFA 切换到 FFA_FA4——Flash-Attention 4 的 monkey patch 版本，以临时支持 Blackwell GPU 上的任意掩码。默认值为 0。

备注

这目前是一个临时解决方案，未来可能会被移除或更新。

MAGI_ATTENTION_NATIVE_GRPCOLL

将此环境变量设为 1 以启用 group collective 通信的原生内核实现。默认值为 0。

备注

此功能目前处于实验阶段并在积极开发中，与许多其他功能不兼容，因此请不要启用，除非您清楚了解其影响。

MAGI_ATTENTION_QO_COMM

将此环境变量设为 1 以启用 query/output 通信，包括获取远程 q（前向）、reduce partial out 和 lse（前向）、获取远程 q,o,lse,do（反向）、reduce partial dq（反向），以消除通信仅限于 key/value 的限制。默认值为 0。

备注

此功能目前处于实验阶段并在积极开发中，与许多其他功能不兼容，因此请不要启用，除非您清楚了解其影响。

MAGI_ATTENTION_FLATTEN_HEAD_GROUPS

将此环境变量设为 1 以展平 GQA/MQA attention 中的 head groups 来优化 dynamic solver 性能。默认值为 0。

备注

此功能目前处于实验阶段并在积极开发中，与许多其他功能不兼容，因此请不要启用，除非您清楚了解其影响。

MAGI_ATTENTION_AUTO_RANGE_MERGE

将此环境变量设为 1 以启用 flex-flash-attention 的自动范围合并，通过减少注意力范围的数量来提升性能。默认值为 0。

备注

此功能目前处于实验阶段并在积极开发中，因此请不要启用，除非您清楚了解其影响。

MAGI_ATTENTION_CATGQA

将此环境变量设为 1 以启用 flex-flash-attention 反向的 CatGQA 模式，通过拼接共享相同 KV head 的多个 Q head 来进一步优化 GQA 设置下的性能。默认值为 0。

备注

此功能目前处于实验阶段并在积极开发中，因此请不要启用，除非您清楚了解其影响。

MAGI_ATTENTION_BWD_HIDE_TAIL_REDUCE

将此环境变量设为 1 以牺牲保存最后一个远程 kv 激活来重排反向传播中的重叠阶段，将最终的远程 group_reduce 隐藏在主机 FFA 阶段中。默认值为 0。

备注

此功能目前处于实验阶段并在积极开发中，与许多其他功能（如 qo comm）不兼容，因此请不要启用，除非您清楚了解其影响。

MAGI_ATTENTION_FFA_FORWARD_SM_MARGIN

设置此环境变量的值以控制 ffa 前向内核为通信内核预留的 SM 数量。当 CUDA_DEVICE_MAX_CONNECTIONS > 1 时默认值为 4，否则为 0。

MAGI_ATTENTION_FFA_BACKWARD_SM_MARGIN

设置此环境变量的值以控制 ffa 反向内核为通信内核预留的 SM 数量。当 CUDA_DEVICE_MAX_CONNECTIONS > 1 时默认值为 4，否则为 0。

MAGI_ATTENTION_CPP_BACKEND

将此环境变量设为 1 以启用核心数据结构（AttnRange、AttnMaskType 等）的 C++ 后端，以避免 Python 开销。默认值为 0。

备注

此功能目前处于实验阶段并在积极开发中。如果未找到 C++ 扩展或此变量设为 0，将回退到 Python 实现。

MAGI_ATTENTION_HIERARCHICAL_COMM

将此环境变量设为 1 以启用二维 cp group（inter_node group + intra_node group）内的分层 group-collective 通信。默认值为 0。

备注

这目前是减少冗余节点间通信的临时方案，未来可能会被移除或更新。

MAGI_ATTENTION_DIST_ATTN_RUNTIME_DICT_SIZE

设置此环境变量的值以控制 dist_attn_runtime_dict_mgr 的最大 LRU 缓存大小。默认值为 1000。

CUDA_DEVICE_MAX_CONNECTIONS

此环境变量定义了 CUDA stream 可以使用的硬件队列数量。增大此值可以改善通信与计算的重叠，但也可能增加 PCIe 流量。默认值为 8。

调试相关#

MAGI_ATTENTION_SANITY_CHECK

将此环境变量设为 1 以启用 magi_attention 内部的大量健全性检查代码。默认值为 0。

备注

此功能仅用于测试或调试，因为额外的健全性检查开销可能不容忽视。

MAGI_ATTENTION_SDPA_BACKEND

将此环境变量设为 1 可将注意力内核后端从 ffa 切换到 sdpa-math，以支持更高精度（如 fp32 或 fp64）。默认值为 0。

备注

此功能仅用于测试或调试，因为其性能不可接受。

MAGI_ATTENTION_DETERMINISTIC_MODE

将此环境变量设为 1 以启用确定性模式，对所有 magi_attention 内核使用确定性算法。默认值为 0。

MAGI_ATTENTION_PROFILE_MODE

将此环境变量设为 1 以启用性能分析模式，对所有 magi_attention 内核进行性能分析，目前主要针对 ffa 内核（详情请参阅此处）。默认值为 0。

备注

此功能仅用于开发，请勿在生产环境中启用。

构建相关#

JIT#

MAGI_ATTENTION_WORKSPACE_BASE

指定 MagiAttention 工作空间的基础目录，包含缓存和生成的源文件。如果未设置，默认为用户主目录（~）。

MAGI_ATTENTION_BUILD_VERBOSE

将此环境变量设为 1 以启用 JIT 编译过程中的详细输出，显示正在执行的完整 ninja 构建命令。默认值为 0。

MAGI_ATTENTION_BUILD_DEBUG

将此环境变量设为 1 以启用 C++/CUDA 编译器的调试标志。这包括 -g（调试符号）等选项和其他用于获取更详细信息（如寄存器使用情况）的标志。默认值为 0。

MAGI_ATTENTION_NO_BUILD_CACHE

将此环境变量设为 1 以禁用已构建 ffa 内核的缓存。默认值为 0。

MAGI_ATTENTION_FORCE_JIT_BUILD

将此环境变量设为 1 以强制以 JIT 模式构建 FFA，即使预构建的 AOT libs 已存在。默认值为 0。

NVCC_THREADS

设置 nvcc 的 --split-compile 选项的线程数，可加速 CUDA 内核的 JIT 编译。默认值为 4。

AOT#

MAGI_ATTENTION_PREBUILD_FFA_JOBS

设置此环境变量的值以控制预构建 ffa 内核时使用的并行编译任务数。默认值为可用 CPU 核心数的 90% 上取整（即 ceil(num_cpu_cores * 0.9)）。

MAX_JOBS

设置此环境变量的值以控制构建 ffa 以外的扩展模块时使用的并行编译任务数。默认值为可用 CPU 核心数的 90% 上取整（即 ceil(num_cpu_cores * 0.9)）。

MAGI_ATTENTION_PREBUILD_FFA

将此环境变量设为 1 以启用对一些 ref_block_size=None 的常见选项预构建 ffa 内核，其余选项则以 JIT 模式构建。默认值为 1。

MAGI_ATTENTION_SKIP_MAGI_ATTN_EXT_BUILD

将此环境变量设为 1 可跳过构建 magi_attn_ext。默认值为 0。

MAGI_ATTENTION_SKIP_MAGI_ATTN_COMM_BUILD

将此环境变量设为 1 可跳过构建 magi_attn_comm。默认值为 0。

MAGI_ATTENTION_FFA_FA4_CACHE_DIR

设置此环境变量以指定预编译 FFA_FA4 内核的缓存目录。默认值为 magi_attention/lib/ffa_fa4_cache/。

MAGI_ATTENTION_BUILD_COMPUTE_CAPABILITY

设置此环境变量以指定构建 MagiAttention 扩展模块时使用的计算能力（影响 magi_attn_comm 和 create_block_mask）。支持逗号分隔的多架构值，例如 MAGI_ATTENTION_BUILD_COMPUTE_CAPABILITY=90,100。如果未设置，我们将尝试检测当前 GPU 设备的计算能力。如果检测失败将抛出错误。

MAGI_ATTENTION_ALLOW_BUILD_WITH_CUDA12

将此环境变量设为 1 以允许使用 CUDA-12 构建 MagiAttention 扩展模块，这可能导致与 CUDA-13+ 相比显著的性能下降。默认值为 0，在此情况下，如果 CUDA 版本低于 13.0，我们将抛出错误并中止安装。

NVSHMEM_DIR

设置此环境变量为自定义 nvshmem 安装目录的路径。

如果未设置，默认查找 requirements.txt 中列出的系统模块 nvidia-nvshmem-cu12。

如果在任何位置都未找到，native group collective 通信内核中的所有相关功能将被禁用。
